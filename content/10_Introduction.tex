\section{Introduction}

\subsection{Overview}
Cloud computing is a revolution in application development and scalability. With the need of modern applications to be highly available and scalable, many enterprises have turned to containerized applications for highly agile and secure deployments. In order to manage these containerized applications, orchestration tools like Kubernetes have become essential. Kubernetes provides a robust platform for automating deployment, scaling, and operations of application containers across clusters of hosts.

Containers are a lightweight software package that contains all the necessary components to run an application, including the code, runtime, system tools, libraries, and settings.
\parencite{docker:containers}
This encapsulation allows for consistent environments across development, testing, and production stages. In order to fulfill the requirements of high availability and scalability, containers are designed to be ephemeral and stateless, which means they can be easily created, destroyed, and replaced without affecting the overall application state. Thus, containers are stateless in nature and are designed to be transient and easily replaceable, should issues with the application arise.
\parencite{containers:design-principles}
This property introduces challenges when it comes to managing stateful applications, such as databases, within a containerized environment. Stateless applications - such as web servers and API gateways - are well-suited for containerization, as they can be easily scaled and managed without the need for persistent storage.
